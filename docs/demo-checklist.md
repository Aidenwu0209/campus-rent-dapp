# 课堂演示流程清单

## 演示前准备

1. 确认本机已安装 Node.js、npm、MetaMask。
2. 在根目录执行 `npm install`。
3. 在 `Front` 目录执行 `npm install`。
4. MetaMask 添加 Ganache 本地网络：
   - RPC URL：`http://127.0.0.1:7545`
   - Chain ID：`1337`
   - Currency Symbol：`ETH`
5. 准备两个 Ganache 测试账户：
   - 账号 A：物品发布者
   - 账号 B：租赁者

## 启动和部署

1. 终端 A 启动 Ganache：

   ```bash
   npm run ganache
   ```

2. 终端 B 编译合约：

   ```bash
   npm run compile
   ```

3. 终端 B 部署合约：

   ```bash
   npm run migrate
   ```

4. 确认迁移脚本已更新：
   - `Front/src/contracts/CampusRental.json`
   - `Front/src/contracts/campusRentalAddress.json`

5. 启动前端：

   ```bash
   cd Front
   npm run dev
   ```

6. 浏览器打开 `http://127.0.0.1:5173/`。

## 页面演示主流程

1. MetaMask 切换到 Ganache Chain ID 1337。
2. 使用账号 A 连接钱包。
3. 顶部确认显示：
   - 钱包地址
   - Chain ID `1337`
   - 余额
   - 合约地址
4. 进入“发布物品”，发布示例物品：
   - 名称：校园充电宝
   - 描述：适合图书馆和教学楼临时使用
   - 日租金：`0.01 ETH`
   - 押金：`0.05 ETH`
   - 最大租赁天数：`7`
5. 回到“物品列表”，确认状态 Badge 为“可租赁”。
6. 切换 MetaMask 到账号 B。
7. 账号 B 在首页选择租赁天数 `2`，点击“租赁”，支付 `0.07 ETH`。
8. 确认物品状态变为“已租赁”。
9. 进入“我的租赁”，确认出现租赁记录。
10. 账号 B 点击“申请归还”。
11. 确认租赁状态变为“待确认归还”。
12. 切换 MetaMask 到账号 A。
13. 进入“我的发布”，点击“确认归还”。
14. 确认：
    - 物品状态恢复为“可租赁”
    - 租赁记录状态为“已完成”
    - 合约退回押金给账号 B
    - 合约结算租金给账号 A

## 上下架演示

1. 账号 A 发布一个未出租物品。
2. 在“我的发布”点击“下架”。
3. 确认状态为“已下架”，该物品不可租赁。
4. 点击“重新上架”。
5. 确认状态恢复为“可租赁”。

## 错误网络演示

1. MetaMask 切换到非 1337 网络。
2. 顶部应出现明显提示。
3. 页面交易按钮应禁用，防止误操作。
4. 切回 Ganache Chain ID 1337 后恢复正常。

## 测试演示

课堂最后可运行：

```bash
npm test
```

重点说明测试覆盖：

- 发布参数校验
- 正确租赁和错误支付
- 重复租赁防护
- 发布者不能租赁自己物品
- 申请归还和确认归还权限
- 押金退回和租金结算
- 下架、重新上架和下架后不可租赁
