# 他人本地 Ganache 体验指南

本文档用于指导其他同学或老师在自己的电脑上运行本项目。每个人都使用自己电脑上的 Ganache、本地测试账户和 MetaMask，不需要使用项目作者的 MetaMask 密码、账户或私钥。

## 一、先理解运行方式

本项目默认使用本地 Ganache：

- RPC：`http://127.0.0.1:7545`
- Chain ID：`1337`
- 合约由运行者自己部署到自己的本地链
- MetaMask 使用运行者自己导入的本地测试账户

`127.0.0.1` 代表“当前这台电脑”。因此，别人 clone 仓库后，需要在他自己的电脑上启动 Ganache 并重新部署合约，不能直接复用你电脑上的本地链状态或合约地址。

## 二、需要准备的软件

运行者电脑需要安装：

- Node.js 和 npm
- Chrome 或 Edge 浏览器
- MetaMask 浏览器插件
- Git

MetaMask 可以使用运行者自己的插件和密码。密码只用于解锁他自己浏览器里的 MetaMask，不需要项目作者提供任何密码。

## 三、获取项目代码

打开终端，执行：

```bash
git clone https://github.com/Aidenwu0209/campus-rent-dapp.git
cd campus-rent-dapp
```

仓库里包含完整源码、合约、测试、前端和文档，但不会包含：

- `node_modules`
- 作者的 MetaMask 账户
- 作者的私钥或助记词
- 作者电脑上的 Ganache 本地链状态
- 作者本机的 `.env`

这些内容本来就不应该提交到仓库。

## 四、安装依赖

在项目根目录安装 Truffle/Ganache 相关依赖：

```bash
npm install
```

再安装前端依赖：

```bash
cd Front
npm install
cd ..
```

## 五、启动本地 Ganache

打开第一个终端，进入项目根目录，执行：

```bash
npm run ganache
```

这个终端需要一直保持运行。看到类似下面的信息，说明本地链已经启动：

```text
RPC Listening on 127.0.0.1:7545
Chain Id: 1337
Available Accounts
Private Keys
```

本仓库的 `npm run ganache` 已使用固定账户模式启动 Ganache：

```bash
--wallet.deterministic
```

这表示每次用本仓库命令启动 Ganache，都会生成同一批本地测试账户。第一次把账户 A、账户 B 导入 MetaMask 后，只要不重置 MetaMask、不卸载浏览器、不手动删除导入账户，后续演示一般不需要重新导入账户。

Ganache 会显示 10 个测试账户和对应私钥。这些私钥只属于当前这台电脑上的临时本地测试链，只能用于课程演示，不要用于真实资产钱包。

需要注意：固定账户只保证“账户地址和测试私钥稳定”。如果关闭并重新启动 Ganache，本地链状态仍会重置；已经发布的物品、租赁记录和旧合约状态不会自动保留，需要重新部署合约并重新演示发布流程。

## 六、添加 MetaMask 本地网络

打开 MetaMask，添加一个自定义网络：

```text
网络名称：Ganache Local 1337
RPC URL：http://127.0.0.1:7545
Chain ID：1337
货币符号：ETH
```

添加后，切换到这个网络。

如果 MetaMask 已经有旧的 Ganache 网络，但页面仍提示网络错误，建议删除旧网络后按上面的参数重新添加。

## 七、导入两个本地测试账户

为了完整演示“发布者不能租赁自己的物品”，至少需要两个账户：

- 账户 A：发布物品、下架、确认归还
- 账户 B：租赁物品、申请归还

导入方式：

1. 回到运行 `npm run ganache` 的终端。
2. 在 `Private Keys` 区域复制第 1 个或第 2 个测试私钥。
3. 打开 MetaMask。
4. 点击账户头像或账户菜单。
5. 选择“导入账户”。
6. 粘贴从自己 Ganache 终端复制的测试私钥。
7. 重复一次，再导入第二个测试账户。

注意：

- 因为 Ganache 已经固定账户启动，账户 A、账户 B 通常只需要第一次导入 MetaMask。
- 后续重新打开项目时，如果 MetaMask 里还保留这两个导入账户，直接切换到 `Ganache Local 1337` 网络即可看到测试余额。
- 不要导入项目作者的真实账户。
- 不要把自己的真实钱包私钥粘贴到项目或终端里。
- 不要把 Ganache 打印的测试私钥提交到 GitHub。
- 如果只是体验功能，使用 Ganache 测试账户即可。

## 八、编译并部署合约

打开第二个终端，进入项目根目录，执行：

```bash
npm run compile
npm run migrate
```

`npm run migrate` 会把合约部署到运行者自己的 Ganache 本地链，并更新前端需要读取的合约信息：

```text
Front/src/contracts/CampusRental.json
Front/src/contracts/campusRentalAddress.json
```

如果关闭并重新启动 Ganache，本地链会重置，合约地址也可能变化。账户 A、账户 B 不需要重新导入，但需要重新执行：

```bash
npm run migrate
```

## 九、启动前端页面

打开第三个终端，进入项目前端目录：

```bash
cd Front
npm run dev
```

浏览器打开 Vite 显示的地址，通常是：

```text
http://127.0.0.1:5173/
```

连接 MetaMask 后，页面顶部应显示：

- MetaMask 已连接
- 当前账户地址
- 网络 `1337`
- 当前账户余额
- 当前合约地址

如果页面提示“请切换到 Ganache Chain ID 1337”，说明 MetaMask 当前网络不正确，需要切换到第六步添加的本地网络。

## 十、完整体验流程

### 1. 使用账户 A 发布物品

在 MetaMask 切换到账户 A。

进入“发布物品”，填写示例：

```text
物品名称：校园充电宝
物品描述：适合图书馆和教学楼临时使用
日租金：0.01
押金：0.05
最大租赁天数：7
```

点击发布并在 MetaMask 中确认交易。

发布成功后，回到“物品大厅”，应看到该物品状态为“可租赁”。

### 2. 使用账户 B 租赁物品

在 MetaMask 切换到账户 B，并刷新页面或点击页面顶部“刷新”。

在“物品大厅”找到刚才账户 A 发布的物品，输入租赁天数，例如：

```text
租赁天数：2
```

页面会显示应付金额：

```text
0.01 ETH * 2 天 + 0.05 ETH 押金 = 0.07 ETH
```

点击租赁，并在 MetaMask 中确认交易。

租赁成功后，物品状态应变为“已租赁”。

### 3. 账户 B 申请归还

保持 MetaMask 为账户 B。

进入“我的租赁”，找到对应租赁记录，点击“申请归还”，并在 MetaMask 中确认交易。

成功后，租赁记录和物品状态应变为“待确认归还”。

### 4. 账户 A 确认归还

在 MetaMask 切换回账户 A，并刷新页面或点击“刷新”。

进入“我的发布”，找到该物品，点击“确认归还”，并在 MetaMask 中确认交易。

确认后：

- 押金退回账户 B
- 租金结算给账户 A
- 合约余额归零
- 物品恢复为“可租赁”

### 5. 演示上下架

保持 MetaMask 为账户 A。

进入“我的发布”，对一个可租赁物品点击“下架”。下架后该物品不会再作为可租赁物品被租赁。

如页面提供“重新上架”操作，可再点击重新上架，使物品恢复为“可租赁”。

## 十一、运行自动测试

在 Ganache 保持运行的情况下，回到项目根目录执行：

```bash
npm test
```

测试通过时会看到类似：

```text
17 passing
```

这些测试覆盖：

- 发布物品成功
- 非法参数发布失败
- 正确金额租赁成功
- 金额错误租赁失败
- 重复租赁失败
- 发布者不能租赁自己的物品
- 非租赁者不能申请归还
- 租赁者申请归还成功
- 非发布者不能确认归还
- 发布者确认归还成功
- 押金退款和租金结算
- 下架物品
- 已下架物品不可租赁

## 十二、常见问题

### 只能连接钱包，但按钮不能操作

通常是以下原因之一：

- MetaMask 没切到 `Chain ID 1337`
- 当前账户不是正确角色，例如发布者不能租自己的物品
- 当前账户没有 Ganache 测试 ETH
- 没有先执行 `npm run migrate`
- Ganache 重启后合约地址变化，但前端没有重新部署更新

处理方式：

```bash
npm run migrate
```

然后刷新前端页面，确认顶部网络和合约地址正常。

### 页面显示合约地址，但没有物品

这是正常的。每个人的 Ganache 本地链都是新的空链，需要先用账户 A 发布一个物品。

### MetaMask 有账户，但余额是 0

说明当前账户不是 Ganache 生成的测试账户。请按第七步导入本机 Ganache 终端显示的测试私钥。

### 交易一直失败或卡住

可以按顺序检查：

1. Ganache 终端是否仍在运行。
2. MetaMask 网络是否为 `Ganache Local 1337`。
3. 前端顶部 Chain ID 是否为 `1337`。
4. 是否重启过 Ganache；如果重启过，重新执行 `npm run migrate`。
5. 当前账户角色是否正确：账户 A 发布/确认，账户 B 租赁/申请归还。

### 能不能直接用作者的 MetaMask 密码

不能，也不需要。

运行者使用自己的 MetaMask 插件和密码即可。课程演示只需要导入自己本机 Ganache 生成的测试账户。
